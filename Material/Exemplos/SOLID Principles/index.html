Princípios SOLID Demonstrados no Game

S - Single Responsibility

Cada classe tem uma única responsabilidade: Player, Bullet, Enemy

O - Open/Closed

Novos projéteis e inimigos podem ser adicionados sem modificar código existente

L - Liskov Substitution

Diferentes tipos de projéteis podem substituir uns aos outros

I - Interface Segregation

Interfaces específicas: Drawable, Movable, Collidable

D - Dependency Inversion

Game depende de abstrações, não de implementações concretas
